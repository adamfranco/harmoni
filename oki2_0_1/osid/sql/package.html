SQL Open Service Interface Definition.

The SQL Open Service Interface Definition (OSID) provides a means of storing, editing, and retrieving tabular data through the execution of SQL statements. The data store is presumed to be a relational database. The SQL OSID provides an additional layer of abstraction between the application and the database enables a SQL OSID implementation:

  • to check user authorization
  • to log activity
  • to execute the SQL OSID methods on a remote machine

A Connection is used to access a database. The Connection provides methods to execute both queries that return ResultTables, and updates that use SQL statements to manage the database's content and structure. The Connection also has a method to return the SQLTypes that are supported by an SQL OSID implementation.

A ResultTable is returned by a Connection query. The ResultTable contains tabular data. The tabular data is arranged in Rows and each Row is composed of columns. The ResultMetaData contains information about a particular column. The ResultTable presents its tabular data as:

  • a RowIterator returned by the getRows method
  • a ResultMetaDataIterator that contains column metadata returned by the getResultMetaData method
  • a count of the number of columns returned by the getColumnCount method

A Row has methods to return all its data with a getColumns method or to return the data for a specific column either by column name or by column index. The column data in a Row can be a Blob, a Clob, or other serializable data.
